This is quite a doozy, as I don't think there exists a way to do this arbitrarily in OOP/Imperative languages either, at least not easily thought of. If we know the number of lists/arrays we need to permute ahead of (compile) time, we can simply write the appropriate number of loops with appropriate end points (which can vary based on the size of each list/array). However, if we don't know the number of lists at compile time (or we want to design a method that's more flexible, say for library purposes), I think we have to go with the functional approach:

Take two lists of arbitrary length, and return each of their permutations as a single list. Then, if we want to permute three or more lists, we simply compose these functions calls as

newlist = permute(a, b)

newmanylists = permute(a, permute(b, permute(c, d)));

Easy enough to do. Now it's just a matter of finding Permutations of two collections
