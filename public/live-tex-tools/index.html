So, this came up in my quest to defeat note taking once and for all. A friend from the r/math discord suggested live tex via laptop in class over using a tablet for handwriting. Both approaches have their merits, although neither directly integrates with Evernote for organization. Ideally, I would use Evernote directly, and simply type prose for notes. That said, I'm still going to want to be able to write math easily while in lecture, so it isn't a totally viable approach. Tex has the advantage of getting everything typeset just right, but Nebo's math recognition from handwriting has the benefit of being slightly faster (only on math!). I'm gravitating toward the keyboard, since it's so much faster for me to enter text that way, but I worry about not being fast enough with tex. Maybe that's something I can fix? I think I should just try both of them, and see how it goes.

I did find a couple of ideas for a template for Cornell Notes in latex, and I'm going to give them a shot too. Then I can just attach pdfs of the typset notes to Evernotes, to keep them organized and searchable. Let's see how it goes.

On to the approach. Writing LaTeX directly is somewhat of a pain, but that pain can be alleviated to some extent via the macros and functions contained in AUCTeX. I'm also considering doing something along the lines of org export to LaTeX, that way I get all of org's nice features like capture and refile and tree organization. I don't have much experience with either, though, so I'll need something to practice on. I'll have to evaluate the pros and cons of writing in a LaTeX buffer directly vs the org buffer and export option. Org export does seem rather painless so far, so it might not be much of a toss up. Finally, there's the option to use LyX instead of emacs and writing buffers directly. I'm going to give that one a shot, too.

https://www.lyx.org/

My laptop has no problem compiling and working with the Cornell notes template I found here https://tex.stackexchange.com/questions/70570/cornell-notes-a-lyx-or-latex-solution-needed, but my desktop does. Right now it's just a problem of getting PDFs to display properly, which I'm working on sorting out right now with PDF-tools. Once I get that configured, I think we'll be set. I forgot how much crap needed to be downloaded on windows to get PDF-tools working!

PDF-tools is now working, and I can view the output of my cornell notes template live in emacs. Sweet! For some odd reason, though, when I typeset it on windows, the summary boxes float over the top of the notes boxes, whereas on linux they both separate themselves and can be clearly seen. Need to dig in, there.
